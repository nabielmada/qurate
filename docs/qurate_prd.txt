1. Executive Summary
Qurate adalah platform pembayaran Web3 berbasis AI Agent yang memungkinkan siapapun membayar dengan crypto hanya dengan scan QR - tanpa perlu memahami token, chain, atau gas fee. AI Agent Qurate secara otomatis menganalisa wallet user, memilih token dan chain paling optimal, mengeksekusi transaksi, dan menjelaskan keputusannya dalam bahasa yang mudah dipahami orang awam.

Core insight: AEON Pay sudah menyelesaikan masalah merchant-side (fiat settlement). Qurate menyelesaikan masalah yang AEON belum selesaikan: user-side intelligence - zero crypto knowledge required, truly autonomous AI Agent yang punya wallet sendiri dan bekerja tanpa intervensi user sama sekali.
Problem Statement
Crypto payment saat ini memiliki UX yang buruk untuk orang awam:
• User harus tahu token apa yang mereka punya dan cocok dengan merchant
• User harus manually approve setiap transaksi di wallet
• User harus mengerti chain mana yang harus dipilih
• Pesan error bersifat teknis dan tidak actionable (misalnya: 'insufficient gas error 0x...')
• Tidak ada yang menjelaskan dalam bahasa manusia kenapa transaksi berhasil atau gagal

Solution
Qurate hadir sebagai lapisan AI di atas infrastruktur Web3 yang ada. User cukup scan QR, konfirmasi nominal, dan AI Agent yang mengerjakan sisanya - pemilihan token, pemilihan chain, kalkulasi gas, konversi, eksekusi, hingga notifikasi merchant.

6. AI Agent Specification
6.1 Agent Architecture
AI Agent Qurate terdiri dari 4 komponen utama yang bekerja secara sequential dalam setiap transaksi:

Komponen 1 - Intent Parser
Membaca dan memvalidasi QR payload. Mengekstrak: merchant ID, nominal IDR, token yang diterima, expiry time, dan transaction ID. Validasi: apakah QR masih berlaku, apakah merchant terdaftar, apakah nominal masuk akal.

Komponen 2 - Wallet Intelligence
Scan wallet user secara real-time menggunakan provider API (Alchemy/Moralis). Mengidentifikasi semua token di semua chain yang didukung. Kalkulasi USD value per token. Filter token yang nilainya cukup untuk membayar (minimal 105% dari nominal untuk cover gas).

Komponen 3 - Route Optimizer
Kalkulasi multi-variabel untuk memilih rute optimal:
Variabel
Logika
Gas fee real-time
Ambil dari gas oracle (Etherscan, Polygonscan, BaseScan) setiap 30 detik
Transaction speed
Estimasi waktu konfirmasi per chain (target < 5 detik untuk L2)
Slippage estimate
Kalkulasi slippage jika perlu swap token via DEX
Token stability
Prioritas stablecoin (USDC/USDT) dibanding volatile asset
Chain preference
Prioritas: Base > Arbitrum > Polygon > BSC > Ethereum
Composite score
Weighted score = (0.4 x gas) + (0.3 x speed) + (0.2 x stability) + (0.1 x liquidity)
Komponen 4 - Natural Language Explainer
Memanggil Claude API (claude-sonnet-4-20250514) dengan konteks transaksi untuk menghasilkan penjelasan yang:
• Ditulis dalam Bahasa Indonesia yang mudah dipahami ibu-ibu pasar
• Menjelaskan KENAPA token dan chain ini yang dipilih
• Memberikan rasa aman: berapa yang dipotong, apakah transaksi aman
• Tidak menggunakan istilah teknis (gas, chain, slippage, mempool, dll)

6.2 AI Decision Logic Flowchart
Urutan keputusan AI Agent per transaksi:
1. Parse QR intent dan validasi expiry
2. Scan semua token di wallet user
3. Filter token yang balance-nya cukup (>= nominal + estimated gas)
4. Jika tidak ada token yang cukup: kembalikan error dengan penjelasan human-friendly
5. Dari token yang eligible, prioritaskan stablecoin
6. Fetch gas price real-time untuk semua chain yang token tersedia
7. Hitung composite score per opsi (token x chain)
8. Pilih opsi dengan score tertinggi
9. Panggil Claude API untuk generate penjelasan
10. Build dan sign transaksi (atau trigger smart contract)
11. Submit ke network dan tunggu konfirmasi
12. Kirim webhook ke merchant dan update UI user

6.3 Claude API Prompt Template
System: Kamu adalah AI payment agent bernama Qurate. Tugasmu menjelaskan keputusan pembayaran crypto dalam bahasa yang mudah dipahami orang awam Indonesia. Jangan gunakan istilah teknis. Maksimal 3 kalimat pendek.
User context yang disertakan: wallet balance, token yang dipilih, chain yang dipilih, gas fee, alasan pemilihan, dan nominal transaksi.

9.8 Antisipasi Pertanyaan Juri
Pertanyaan
Jawaban
Bedanya apa sama AEON?
AEON adalah infrastruktur merchant. Qurate adalah intelligence layer. Di AEON, user masih harus pilih token dan chain sendiri, dan approve setiap transaksi secara manual. Qurate fully autonomous - zero input dari user selain tombol bayar.
AI-nya genuine atau gimmick?
Claude API dipanggil real-time di setiap transaksi. Penjelasannya kontekstual - berbeda untuk setiap kombinasi wallet, token, dan merchant. Bukan template, bukan hardcoded.
Legal di Indonesia?
Untuk demo dan MVP, kami fokus ke cross-border use case yang jauh lebih clear secara regulasi. Untuk Indonesia domestik, kami framing sebagai konversi instan (crypto ke IDR) bukan pembayaran crypto langsung.
Revenue model-nya apa?
0.1% fee per transaksi (jauh lebih murah dari exchange fee 0.3-0.5%). Merchant subscription untuk dashboard premium dan AI analytics. B2B API untuk developer yang mau embed Qurate di app mereka.
Apakah bisa production dalam 4 minggu?
MVP untuk demo dan hackathon: yes. Production-grade dengan smart contract audit, KYC, dan lisensi: 6-12 bulan. Kami transparent tentang ini.
Kenapa tidak launch token sendiri?
Sesuai aturan hackathon Four.meme, tidak wajib issue token. Business model kami fee-based, lebih sustainable dari token model. Token bisa dipertimbangkan post-hackathon jika ada alasan product yang kuat.
